# Security policy

## Supported versions

Security updates are applied on the default development branch of this repository. Use the latest release or main branch for deployments.

## Reporting a vulnerability

Please **do not** open a public GitHub issue for security reports.

Instead, contact the maintainers privately:

- Use [GitHub private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability) if it is enabled for this repository, **or**
- Email or message the repository owners through the contact channel they publish on the project homepage or organization profile.

Include enough detail to reproduce the issue (affected component, steps, impact) and, if possible, suggested remediation. You should receive an acknowledgment within a reasonable time frame; severity and patch timeline depend on maintainer availability.

## Scope

In scope: this application codebase, configuration shipped in-repo, and documented deployment assumptions (Next.js app, database, auth, AI provider credentials).

Out of scope: third-party services unless the issue is directly caused by how this repo integrates with them.
